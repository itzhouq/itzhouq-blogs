# Hermes Agent 视频口播文案

---

## 第1页：封面

哈喽大家好，今天给大家介绍一个非常有意思的 AI Agent 项目——Hermes Agent。

它来自 Nous Research，2026 年 2 月发布，几个月时间就在 GitHub 上拿到了 28,000 多颗星。

它最大的特点只有一个：**会自己学习**。

传统 AI Agent 每次对话从零开始，但 Hermes 不一样，它会从经验中成长，越用越懂你。

---

## 第2页：传统 Agent 的局限

在说 Hermes 之前，我们先看看传统 AI Agent 存在哪些问题。

**第一，每次对话从零开始。** 你昨天教它的东西，今天就忘了，因为它根本没有记忆。

**第二，重复完成相似任务。** 同样是整理会议纪要，它每次都要重新理解你的格式要求，不能把经验沉淀下来。

**第三，无法理解用户偏好。** 你喜欢用列表还是表格，它永远不知道，每次都要重新说明。

这三个问题，本质上都是因为：传统 Agent 没有学习能力。

---

## 第3页：Hermes 的核心定位

那 Hermes Agent 是怎么解决这些问题的呢？

它有四大核心定位。

**第一，内置学习闭环。** 它不是简单地"记住"你说过什么，而是真正从经验中提取模式、持续进化。

**第二，自动创建技能。** 当你完成一个复杂任务，它会自动把这个经验提炼成可复用的技能，下次直接调用。

**第三，跨平台统一。** 一个 Agent，多个入口，命令行、Telegram、Discord、Slack，想在哪用就在哪用。

**第四，隐私友好。** 可以完全本地部署，你的数据不上云。

---

## 第4页：自我学习闭环

这是 Hermes 最核心的特性，我重点讲一下。

它的学习闭环分为四个步骤：

**第一步，任务执行。** 当你让它完成一个复杂任务，涉及 5 个以上的工具调用时，它就开始工作了。

**第二步，模式提取。** 任务完成后，它会自动分析这次工作的流程，识别出可以复用的模式。

**第三步，技能存储。** 提炼出来的技能会保存在本地，下次遇到类似场景直接调用。

**第四步，持续优化。** 每次执行后，它会根据效果自动改进这个技能，让它变得越来越好用。

这个循环，就是 Hermes 能够"越用越聪明"的秘密。

而且，用户纠正错误后也会触发技能创建。它还支持对技能进行 patch 和 edit，随时可以手动优化。

---

## 第5页：记忆系统

除了技能，Hermes 还有一套双文件记忆架构。

第一个文件叫 **MEMORY.md**，存储 Agent 的通用记忆，跨会话积累的知识。

第二个文件叫 **USER.md**，存储你这个用户的个性化信息，包括你的偏好、习惯和特殊需求。

两个文件都在本地存储，路径是 ~/.hermes/memories/

它还支持 LLM 自动总结关键内容，以及跨会话搜索历史对话。

---

## 第6页：用户建模

除了通用记忆，Hermes 还使用一个叫 **Honcho** 的工具来进行用户建模。

简单来说，就是通过对话的方式，持续理解你这个用户是谁、你喜欢什么、你的工作习惯是什么。

它不是一次性记住，而是不断更新、持续学习。

这样服务你的时间越长，它就越懂你，真正成为一个懂你的 AI 伙伴，而不是一个通用工具。

---

## 第7页：跨平台支持

Hermes 支持非常多的平台，我给大家列一下：

**命令行**，有完整的 TUI 界面，支持多行编辑、流式输出。

**Telegram**，支持语音转文字，随时随地对话。

**Discord 和 Slack**，适合社区和企业协作。

还有 **Email**、**WhatsApp**、**Signal** 等等。

关键点是：不管你在哪个平台，只需要运行一个 gateway 进程，就能访问同一个 Agent。

---

## 第8页：多模型支持

 Hermes 不绑定任何模型提供商。

它支持 OpenRouter 上的 200 多个模型，OpenAI 的 GPT 系列，Anthropic 的 Claude 系列，还有国产的 GLM、Kimi、MiniMax 等等。

切换模型也非常简单，一行命令就搞定：

hermes model --provider openrouter --model claude-3.5-sonnet

这种开放性，让你可以根据不同场景选择最合适的模型，不用被单一服务商锁定。

---

## 第9页：部署方式

Hermes 的部署方式非常灵活。

**本地运行**，直接在笔记本上跑就行。

**Docker**，容器化部署，一键启动。

还有 **Daytona、Singularity、Modal** 等云服务，其中 Modal 空闲时近乎零成本。

最低配置，只需要一个 **5 美元的 VPS**，当然如果需要更强性能，也可以扩展到 GPU 集群。

---

## 第10页：快速开始

想试试 Hermes？三步就够了。

**第一步，安装。** 克隆仓库，然后 pip install 安装依赖。

**第二步，配置。** 运行 hermes init 初始化，设置你的 API 密钥。

**第三步，运行。** 直接 hermes run 启动，也可以指定模型和提供商。

整个过程不超过 5 分钟。

---

## 第11页：与 OpenClaw 对比

可能有观众会问：Hermes 和 OpenClaw 有什么区别？我来做一个简单对比。

**学习能力方面**，Hermes 有内置学习闭环，OpenClaw 没有。

**技能创建方面**，Hermes 支持自动创建，OpenClaw 不支持。

**记忆持久化方面**，Hermes 有双文件架构，OpenClaw 没有。

**多平台方面**，两者都支持多种平台。

**多模型方面**，Hermes 更开放，支持各种提供商，OpenClaw 主要面向 Claude。

**多 Agent 协作方面**，这是 OpenClaw 的核心定位，Hermes 目前没有。

总结一下：如果你注重 AI 的自学习能力，Hermes 更强；如果你需要多 Agent 协作，OpenClaw 更适合。

两者其实是互补的关系，可以根据场景选择。

---

## 第12页：优缺点分析

客观来说，Hermes 也有一些不足。

**优点方面**：真正的自学习能力、MIT 完全开源、隐私友好可本地部署、不锁定模型提供商、跨平台统一入口。

**缺点方面**：学习曲线需要一定时间、完整功能需要一定硬件配置、多平台配置比较复杂、相比 Claude Code 生态还比较新。

---

## 第13页：总结

好，最后总结一下。

Hermes Agent 代表了 AI Agent 的一个新方向：从"工具"到"伙伴"的转变。

它的内置学习闭环让 Agent 真正能够从经验中成长，而不仅仅是执行命令。

如果你注重隐私、需要跨平台使用、期望 AI 能够持续学习和进化，Hermes Agent 值得一试。
